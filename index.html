pysasl Documentation¶

exception pysasl.AuthenticationError¶

Indicates that authentication failed due to a protocol error unrelated to any provided credentials.

exception pysasl.UnexpectedAuthChallenge¶

During client-side authentication, the SASL mechanism received an authentication challenge from the server that it did not expect.

class pysasl.AuthenticationCredentials(authcid, secret, authzid=None)¶

Object returned by server_attempt() and passed in to client_attempt() containing information about the authentication credentials in use.

Parameters:
  • authcid – Authentication ID string (the username).
  • secret – Secret string (the password).
  • authzid – Authorization ID string, if applicable.
has_secret¶

True if the secret attribute is valid for this credentials type.

If this returns False, the secret attribute should raise AttributeError.

authcid¶

The authentication identity string used in the attempt.

secret¶

Contains the secret string used in the authentication attempt, if available. Use check_secret() instead, when possible.

authzid¶

The authorization identity string used in the attempt, or None if this field is empty or unused.

identity¶

The canonical identity being assumed by the authentication attempt. This is authzid if available, authcid otherwise.

Consider a UNIX system where root is the superuser and only it may assume the identity of other users. With an authcid of root and an authzid of terry, the authorization would succeed and identity would be terry. With an authcid of greg, authorization would fail because greg is not the superuser and cannot assume the identity of terry.

check_secret(secret)¶

Checks if the secret string used in the authentication attempt matches the “known” secret string. Some mechanisms will override this method to control how this comparison is made.

Parameters:secret – The secret string to compare against what was used in the authentication attempt.
Returns:True if the given secret matches the authentication attempt.

class pysasl.BaseMechanism¶

Base class for all server- and client-side SASL mechanisms.

name¶

The SASL name for this mechanism.

priority¶

Determines the sort ordering of this mechanism.

insecure¶

Whether this mechanism is considered secure for non-encrypted sessions. This value should be used by implementations to determine which mechanisms are chosen or offered.

class pysasl.ServerMechanism¶

Base class for implementing SASL mechanisms that support server-side credential verification.

server_attempt(challenges)¶

For SASL server-side credential verification, receives responses from the client and issues challenges until it has everything needed to verify the credentials.

If a challenge is necessary, an ServerChallenge exception will be raised. Send the challenge string to the client with get_challenge() and then populate its response with set_response(). Finally, append the exception to the challenges argument before calling again.

Parameters:challenges – The server challenges that have been issued by the mechanism and responded to by the client.
Returns:A tuple of the authentication credentials received from the client once no more challenges are necessary, and an optional final response string from the server used by some mechanisms.
Raises:ServerChallenge – The server challenge needing a client response.
class pysasl.ClientMechanism¶

Base class for implementing SASL mechanisms that support client-side credential verification.

client_attempt(creds, responses)¶

For SASL client-side credential verification, produce responses to send to the server and react to its challenges until the server returns a final success or failure.

Send the response string to the server with the get_response() method of the returned ClientResponse object. If the server returns another challenge, set it with the object’s set_challenge() method and append the object to the responses argument before calling again.

Parameters:
  • creds – The credentials to attempt authentication with.
  • responses – The client responses that have been sent to the server. New attempts begin with an empty list.
Returns:

The client response to the most recent server challenge.
Raises:

UnexpectedAuthChallenge – The server has issued an unexpected challenge the client mechanism does not recognize.
class pysasl.SASLAuth(advertised=None)¶

Manages the mechanisms available for authentication attempts.

Parameters:advertised – List of available SASL mechanism objects. Using the name of a built-in mechanism (e.g. b'PLAIN') works as well. By default, all built-in mechanisms are available.
classmethod secure()¶

Uses only authentication mechanisms that are secure for use in non-encrypted sessions.

Returns:A new SASLAuth object.
classmethod plaintext()¶

Uses only authentication mechanisms that provide the credentials in un-hashed form, typically meaning has_secret is True.

Returns:A new SASLAuth object.
server_mechanisms¶

List of available ServerMechanism objects.

client_mechanisms¶

List of available ClientMechanism objects.

get(name)¶

Get a SASL mechanism by name. The resulting object should inherit either ServerMechanism, ClientMechanism, or both.

Parameters:name – The SASL mechanism name.
Returns:The mechanism object or None
get_server(name)¶

Like get(), but only mechanisms inheriting ServerMechanism will be returned.

Parameters:name – The SASL mechanism name.
Returns:The mechanism object or None
get_client(name)¶

Like get(), but only mechanisms inheriting ClientMechanism will be returned.

Parameters:name – The SASL mechanism name.
Returns:The mechanism object or None
