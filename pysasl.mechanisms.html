pysasl.mechanisms Package¶

class pysasl.mechanisms.plain.PlainMechanism¶

Implements the PLAIN authentication mechanism.

server_attempt(responses: Sequence[pysasl.ChallengeResponse]) → Tuple[pysasl.creds.AuthenticationCredentials, None]¶

For SASL server-side credential verification, receives responses from the client and issues challenges until it has everything needed to verify the credentials.

If a challenge is necessary, a ServerChallenge exception will be raised. The response to this challenge must then be added to responses in the next call to server_attempt().

Parameters

responses – The challenge-response exchanges thus far.

Returns

A tuple of the authentication credentials received from the client once no more challenges are necessary, and an optional final response string from the server used by some mechanisms.

Raises

ServerChallenge – The server challenge needing a client response.

client_attempt(creds: pysasl.creds.AuthenticationCredentials, challenges: Sequence[pysasl.ServerChallenge]) → pysasl.ChallengeResponse¶

For SASL client-side credential verification, produce responses to send to the server and react to its challenges until the server returns a final success or failure.

Parameters

  • creds – The credentials to attempt authentication with.

  • challenges – The server challenges received.

Returns

The response to the most recent server challenge.

Raises

UnexpectedChallenge – The server has issued a challenge the client mechanism does not recognize.

class pysasl.mechanisms.crammd5.CramMD5Result(username: str, challenge: bytes, digest: bytes)¶

Because this mechanism uses hash algorithms to compare secrets, the server_attempt() method returns this sub-class which overrides the check_secret() method.

property has_secret: bool¶

True if the secret attribute is valid for this credentials type.

If this returns False, the secret attribute should raise AttributeError.

property challenge: bytes¶

The challenge string issued by the server.

property digest: bytes¶

The digest computed by the client.

property secret: str¶

The secret string is not available in this mechanism.

Raises

AttributeError –

check_secret(secret: Optional[pysasl.creds.StoredSecret], **other: Any) → bool¶

Checks if the secret string used in the authentication attempt matches the “known” secret string. Some mechanisms will override this method to control how this comparison is made.

Parameters

  • secret – The secret to compare against what was used in the authentication attempt.

  • other – Additional keyword arguments to allow subclasses to accept additional data as needed.

Returns

True if the given secret matches the authentication attempt.

Raises

ExternalVerificationRequired – The credentials cannot be verified using the given secret and require external verification.

class pysasl.mechanisms.crammd5.CramMD5Mechanism¶

Implements the CRAM-MD5 authentication mechanism.

Warning

Although secure during transport, offering this mechanism can be dangerous, as it can have implications about how the credentials are stored server-side.

class pysasl.mechanisms.external.ExternalResult(authzid: Optional[str] = None, *, authcid_type: Optional[str] = None)¶

External credentials do not contain authentication credentials, only an identity to authorize as.

property has_secret: bool¶

True if the secret attribute is valid for this credentials type.

If this returns False, the secret attribute should raise AttributeError.

property secret: str¶

Contains the secret string used in the authentication attempt, if available. Use check_secret() instead, when possible.

check_secret(secret: Optional[pysasl.creds.StoredSecret], **other: Any) → NoReturn¶

This implementation does not use secret and instead raises ExternalVerificationRequired immediately.
